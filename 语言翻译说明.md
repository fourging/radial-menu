# 环形菜单模组 - 语言翻译说明

## 概述

感谢您对环形菜单模组的关注！本模组支持多语言本地化，目前已支持简体中文、繁体中文、英语、日语、韩语和俄语。如果您希望为模组添加新的语言支持，请参考以下说明。

## 翻译内容结构

模组的本地化系统基于键值对结构，每个文本项都有一个唯一的键名和对应的翻译文本。所有翻译内容都存储在 `patchs/LocalizationHelper.cs` 文件中。

### 翻译分类

翻译内容主要分为以下几个类别：

1. **设置界面** - 模组设置界面的标题和选项
2. **配置项描述** - 各种设置选项的详细说明
3. **UI提示信息** - 用户界面中显示的各种提示信息
4. **日志信息** - 控制台日志中记录的信息
5. **物品使用相关提示** - 使用物品时显示的提示信息
6. **吃屎相关提示** - 特殊物品（粪便）相关的幽默提示
7. **绑定相关提示** - 物品绑定功能相关的提示信息
8. **输入处理相关提示** - 用户输入操作相关的提示信息
9. **食物绑定相关提示** - 食物自动绑定功能相关的提示信息

## 如何提供翻译

如果您想为新语言提供翻译，请按照以下步骤操作：

### 1. 确定语言代码

首先，请确定您要翻译的语言对应的 `SystemLanguage` 枚举值。Unity 支持的语言包括但不限于：
- `SystemLanguage.French` (法语)
- `SystemLanguage.German` (德语)
- `SystemLanguage.Spanish` (西班牙语)
- `SystemLanguage.Italian` (意大利语)
- `SystemLanguage.Portuguese` (葡萄牙语)
- `SystemLanguage.Polish` (波兰语)
- `SystemLanguage.Dutch` (荷兰语)
- `SystemLanguage.Turkish` (土耳其语)
- 等等...

### 2. 翻译所有键值对

请参考现有的翻译（如英语部分），为每个键提供对应语言的翻译。以下是翻译时需要注意的事项：

#### 翻译原则
- **保持简洁**：UI 文本应尽量简洁，避免过长导致界面显示问题
- **保持风格一致**：保持与游戏整体风格和语调一致
- **保留占位符**：如 `{0}`、`{1}` 等占位符必须保留，它们会被动态替换为实际值
- **文化适应**：适当调整幽默内容，使其符合目标语言的文化背景

#### 特殊注意事项
- **占位符**：确保所有 `{0}`、`{1}` 等占位符在翻译中保留，它们会被替换为实际值
- **格式化**：保持与原文相同的格式，如换行、标点符号等
- **技术术语**：对于游戏特有的术语，请使用目标语言中常见的对应词汇

### 3. 提供完整的翻译数据

请按照以下格式提供完整的翻译数据：

```csharp
// 您的语言
LocalizationData[SystemLanguage.YourLanguage] = new Dictionary<string, string>
{
    // 设置界面
    { "Settings_Title", "您的语言：环形菜单设置" },
    
    // 配置项描述
    { "Config_FoodBindSectors", "自动绑定食物的扇区编号" },
    // ... 其他所有配置项
    
    // UI提示信息
    { "UI_ItemCount", "剩余数量: {0}" },
    // ... 其他所有UI提示
    
    // ... 继续翻译所有剩余的键值对
};
```

## 提交翻译

完成翻译后，请通过以下方式提交：

1. **GitHub**：如果您熟悉 GitHub，可以提交 Pull Request
2. **创意工坊**：在模组的创意工坊页面留言或私信
3. **社区论坛**：在相关社区论坛发布您的翻译

## 测试翻译

在提交翻译前，建议进行以下测试：

1. **完整性检查**：确保所有键值对都已翻译
2. **占位符检查**：确保所有占位符（如 `{0}`）都已正确保留
3. **长度检查**：确保翻译后的文本长度适中，不会导致 UI 显示问题
4. **语法检查**：确保翻译文本语法正确，没有拼写错误
